Les clés pour s'orienter dans un projet libre

Julien Wajsberg, Paris-Web, EfreiTech, 18 octobre 2014

Mozilla a une mission

  • Conserver le web ouvert et accessible.
  • Internet doit améliorer la vie des personnes.
  • La sécurité est fondamentale  ce n'est pas une option.
  • Chacun doit pouvoir faire ses propres expériences sur Internet.

Firefox OS : un OS mobile libre pour le web

  • Les applications sont faites avec le tryptique HTML/CSS/JavaScript
  • Développement libre, mais surtout ouvert (code ouvert en permanence, listes de diffusion, tracker de bug)
  • Cible les mobiles d'entrée de gamme dès le début, avec un focus sur la simplicité pour des nouveaux utilisateurs de smartphones.

Moi et le logiciel libre

  • 1996 : j'accède à Internet
  • libre-antenne et IRC
  • Accès distant à un shell et eggdrops

J'ai, comme tout le monde, utilisé des logiciels gratuits. mIRC pour se connecter sur IRC, WS-FTP, et autres. WarFTPd est sans doute le premier logiciel libre que j'ai utilisé, sans que j'en aie vraiment conscience.

Le premier "vrai" logiciel libre que j'ai utilisé, c'était vraiment l'eggdrop, un bot IRC. Je l'ai compilé moi-même de manière distante sur une machine Linux accédée en SSH.

Qu'est-ce qu'un logiciel libre ?

Historique rapide du libre

  • À l'origine, partage et code source entre hackers
  • Stallman crée la FSF et le projet GNU

Les hackers partageaient leur code et avaient le sens du perfectionnisme. Cela a changé lorsque l'informatique s'est dévelopée : plus de code source, plus de partage, nombre croissant de logiciels privateurs.

Richard Stallman décide de créer la Free Software Foundation pour retrouver les réflexes de partage du début de l'informatique.

Les licences sont des contrats

  • Sans licence, un utilisateur n'a aucun droit sur un logiciel.
  • Un contrat permet de définir les droits accordés à l'utilisateur.
  • Un contrat ne lie que le propriétaire du logiciel et 1 utilisateur.
  • Le logiciel libre détourne ces règles pour donner beaucoup de droits à l'utilisateur.

Stallman et la FSF décident de contourner le principe du copyright en vigueur aux États-Unis (proche du système des droits d'auteur français) et élaborent de véritables contrats. Ainsi, ils se servent des règles normalement contraignantes du copyright pour, au contraire, permettre des droits supplémentaires pour les utilisateurs.

Puisque ce sont des contrats, ils ne s'appliquent effectivement qu'aux destinataires des logiciels.

Des libertés

Un logiciel libre donne des libertés à l'utilisateur

Les 4 libertés

  1. Liberté d'exécuter le programme
  2. Liberté d'étudier le fonctionnement du programme et de l'adapter à ses besoins. Accès au code source.
  3. Liberté de redistribuer des copies.
  4. Liberté de redistribuer des modifications.
Ce sont les 4 règles fondamentales du logiciel libre, édictées par la FSF. Le logiciel libre est un mouvement avec une idéologie : les utilisateurs doivent avoir le contrôle sur leur matériel et le logiciel qui tourne dessus. D'autres membres de la communauté libre s'attachent plutôt aux intérêts techniques du développement libre qui permet d'avoir des logiciels de meilleure qualité.

Du jargon

On utilise beaucoup de jargon dans le monde du logiciel libre.

Du jargon sur le code

  • Bug tracker (suivi des anomalies)
  • Code repository (dépôt de code)
  • Pull request (demande d'intégration de modification)
  • Patch (modification)

La première étape pour avoir un problème corrigé, c'est d'ouvrir un bug. Cela signifie que l'on va aller sur le logiciel de suivi et signaler l'existence d'un problème. Ainsi on va tâcher de donner le plus possible d'informations, et si possible, un exemple minimal qui montre le problème.

Si on veut corriger un problème soi-même, il faut d'abord aller le dire sur le logiciel de suivi, de manière à ce que plusieurs personnes ne bossent pas sur le même problème. On va ensuite récupérer le code du projet pour travailler localement. Lorsque l'on a fini de corriger le problème, tout en créant des tests pour le problème, on va créer un patch que l'on va ajouter sur le bug dans le logiciel de suivi, ou bien créer une pull request.

Du jargon sur la communication

  • Mailing list (liste de diffusion)
  • IRC

Dans un logiciel libre, le développement se fait de manière distribuée. Il faut donc des moyens de communication adaptés.

Dans les logiciels hébergés sur une plate-forme telle que GitHub, beaucoup de communication passe par les commentaires au sein d'une Pull Request.

Les listes de diffusion

  • C'est une adresse e-mail gérée par un logiciel.
  • On s'abonne à une liste.
  • On reçoit ensuite tous les messages envoyés à la liste.
  • Il peut y avoir plusieurs listes pour un logiciel.
  • Les listes de diffusion sont généralements archivées, les archives peuvent ou non être lisibles par les non-abonnés.
  • Certaines listes peuvent avoir plusieurs centaines d'abonnés.

Autrement, beaucoup de logiciels ont aussi un système de liste de diffusion : il suffit de s'y abonner, et on reçoit ensuite tous les mails envoyés sur la liste. Il est alors facile de discuter d'un problème. Attention : il y a parfois plusieurs listes pour un même produit ; certaines sont plutôt orientées développement (et se terminent généralement par -dev), d'autres orientées utilisation (et se terminent par -users).

Les listes de diffusion sont archivées, bien souvent sur Google Groups, ce qui permet une recherche aisée. Il est donc poli de rechercher tout d'abord les archives avant de poser une question, au cas probable où elle aurait déjà été posée et qu'on y aurait déjà répondu dans le passé.

Il ne faut pas oublier que certaines listes peuvent avoir plusieurs centaines d'abonnés. Dès lors, pour ne pas faire perdre de temps à tous, il est important d'avoir un sujet de mail adéquat, et de ne pas envoyer de mails inutiles (type "Moi aussi !").

IRC

  • Assez primitif, existe depuis 1988
  • Système de communication uniquement textuel, plutôt synchrone.
  • En rejoignant un canal de discussion, on peut dialoguer avec les personnes connectées en même temps sur le même canal de discussion.
  • Des discussions privées sont également possibles.
  • Discussions plus informelles que sur une liste de diffusion.
